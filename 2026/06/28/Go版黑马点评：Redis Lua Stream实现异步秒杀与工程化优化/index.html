从“能消费”到“可靠消费”：Pending 与 DLQ

一开始消费者只要读取消息、处理订单、ACK 就可以。

但是如果服务在读到消息后宕机，这条消息就不会被 ACK，会留在 Pending List。

所以消费者启动时要先处理 pending：

这里我没有让 pending 消息在一次循环里疯狂重试到底，而是让 handleMessageWithRetry 返回一个布尔值。

如果这次处理失败，但还没达到最大重试次数，就先返回，等下一轮再处理。

这样可以避免 MySQL 短暂故障时，消息很快被连续重试 3 次然后直接进死信队列。
