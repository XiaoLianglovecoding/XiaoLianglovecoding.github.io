十一、这部分最容易踩的坑

1. 把 authorID 当成 viewerID

如果用主页主人的 ID 去判断点赞状态，就变成：

所以必须拆成 authorID 和 viewerID。

2. Redis 错误不能全部当成“没点赞”

redis.Nil 才表示 member 不存在，也就是没点赞。

其他错误可能是 Redis 异常，应该打日志或返回失败。

3. MySQL 和 Redis 不是一个事务

现在用了补偿回滚，但它仍然不是强一致。

这不是坏事，反而是学习项目里很重要的一个阶段：先理解问题，再逐步优化。

4. 公开接口是“可选登录”

GET /blog/hot、GET /blog/:id、GET /blog/of/user 都应该允许游客访问。

所以这些接口不能使用 LoginInterceptor 强制拦截。

但是如果请求头里带了合法 token，就应该返回正确的 isLike。

这就是全局 RefreshTokenMiddleware 的价值。
