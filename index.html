Go 版黑马点评：从零搭建后端项目骨架

这篇文章记录我用 Go 重写黑马点评后端的第一天：不急着写登录、点赞、秒杀这些业务，而是先把一个后端项目最基础的骨架搭起来。

这一阶段的目标很明确：

  • Go 后端独立成一个新项目，不污染原来的 Java 后端
  • 使用 Gin 搭 HTTP 服务
  • 使用 Viper 读取配置文件
  • 使用 GORM 预留 MySQL 访问能力
  • 使用 go-redis 预留 Redis 客户端
  • 按 Handler、Service、Repository 分层
  • 用统一 Result 返回结构对齐前端
  • 先把前端会访问的路由注册出来，没实现的接口先返回 TODO
